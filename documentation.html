Start Page

The start page introduce the players in the world of the game. They'll be presented some snippets from inside the game and some information regarding the way they are driven through the HTTP protocol. Then, there are presented some of the benefits learning through games has compared to conventional methods.

The user have three buttons available in the menu bar.

  • Home - redirects the user to the home page
  • Create Account - puts the base for going "online" by creating an account and a storage for the user's personal score
  • Log In - the user goes "online" and so it is enhanced his experience (the game can also be played offline but no personal score is stored)

Create Account

To create an account, the user must fill in the three fields, respecting the film format requirements, and then press Register. If everything is ok, the user is notified that the account was created successfully and is redirected towards the log in page.

Log in

To log in, the user must fill in the fields required with the personal info he introduced to create the account.

Levels Page

In this page the user can see the levels bar. He can play a new level as long as it is available (colored) to unlock new cool levels (grey). He can also go back to a previously completed level if he wants to exceed his personal score.

Down left, the user's total score is displayed. He has diamonds, special artifacts that can be collected exclusively on the map, and stars which are awarded by answering correct to the HTTP questions.

Down right, there is a menu containing three buttons:

  • Home - redirects the user to the home page
  • Profile - a mini personal page where the user can view his score, can change his username, email or profile picture
  • Log Out - the user is disconnected from the application and redirected to the home page

Inside the game

After the user selects the level he wants to play, the game begins. The user is guided throughout the game through paper scrolls that form the story line and contain all the necessary information for the user to understand how the game works and what he has to do to complete a level.

The personal score is also displayed in this page in the upper left corner of the screen along with the button that redirects the user towards the levels page. Be careful because when you return to the levels page, you lose the current progress within the level.

For each level, the user will overcome obstacles (monsters, events, etc.) which will engage him with trivia-type questions and also clues (possible false). When a question appears, the user must select the answer he believes it's right and the press Submit. After that, the correct answer wink and the user is notified whether he answered correctly or not. Then he must press Next so the game continues.

Conclusion

As you can see after you finish the third level, the game is still work in progress; more levels will be available in the future with new topics from the HTTP theory.

We hope that the game, in the way it was conceived, represents a solid foundation for learning and developing the notions around the HTTP protocol area.
